# 04-比赛执行方案

## 1. 比赛基本信息

- **比赛名称**: OKX AI 交易大赛
- **计榜要求**: 通过 Agent Trade Kit 自动执行的 USDT 永续合约交易
- **标的**: ETH-USDT-SWAP（USDT 永续合约）
- **排行榜**: 按收益额排名

## 2. 核心参数

```json
{
  "strategy": "ETH 震荡网格",
  "symbol": "ETH-USDT-SWAP",
  "exchange": "OKX",
  
  "grid": {
    "type": "等差",
    "count": 8,
    "price_min": 1900,
    "price_max": 2200
  },

    "leverage": 2,
    "margin_mode": "逐仓",
    "max_position": "0.24 ETH",
    "stop_loss": "-5%",
    "take_profit": "每格 1.86%"
  },
  
  "execution": {
    "entry_condition": "震荡得分 > 60%",
    "exit_condition": "突破区间边界 3% 或连续 3 天无套利",
    "rebuild_condition": "价格持续在区间内 7 天"
  }
}
```

## 3. Agent Trade Kit 接入

### 需要准备的文件
- `SKILL.md` - 上传到 Agent Trade Kit 的主文件
- 历史证据（已整合在 skill/ 目录）
- 参数方案（见上文 JSON）

### 执行流程
```
Claude Code (本地)
  ↓ 生成/更新 SKILL.md
Agent Trade Kit
  ↓ 读取 SKILL.md，自动执行订单
OKX 账户
  ↓ 持仓、成交、收益
排行榜
```

## 4. 监控指标

| 指标 | 正常范围 | 告警阈值 |
|------|----------|----------|
| 日往返次数 | 2-6 次 | <1 次或 >10 次 |
| 持仓时间 | <24 小时 | >48 小时 |
| 未匹配买单 | 0-4 笔 | >6 笔 |
| 当日盈亏 | -5% ~ +5% | <-5% |

## 5. 应急预案

### 场景 1: 价格突破上沿
- 暂停新买单
- 评估是否上移区间或止盈退出
- 24 小时内决策

### 场景 2: 价格突破下沿
- 暂停网格
- 评估是否下移区间或止损
- 12 小时内决策

### 场景 3: 连续无套利
- 检查市场状态
- 如震荡得分 < 50%，暂停策略
- 等待震荡恢复

## 6. 免责声明

本策略基于历史 ETH-USDT 现货网格样本分析，迁移至 ETH-USDT-SWAP 永续合约后：
- 杠杆会放大收益和亏损
- 资金费率会影响长期持仓成本
- 历史表现不代表未来收益
- 请自行评估风险
